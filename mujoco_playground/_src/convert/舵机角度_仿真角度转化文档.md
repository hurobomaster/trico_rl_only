## 核心转换逻辑

### Real2Sim：真机舵机 → 仿真Hinge角（观测）

```
真机舵机 [θ51, θ52, θ53, θ54, θ61, θ62, θ63, θ64]°
    ↓ [Step 1: FK正运动学]
方向向量 [OA_right, OB_right, OA_left, OB_left] (单位向量)
    ↓ [Step 2: 坐标系偏置补偿]
      (VirtualHingeMapper 应用 OA:121.6°, OB:211.6° 偏置)
仿真Hinge角 [θOA_x_r, θOA_y_r, θOB_x_r, θOB_y_r, ...] (弧度)
    ↓ [Step 3: 零位校准]
      (减去初始HOME位置的偏置)
相对Hinge角 → RL网络输入
```

**关键点**：
- FK 输出必须是单位向量 (模 ≈ 1.0)
- VirtualHingeMapper 处理坐标系偏置，不需手动计算
- 初始HOME位置的偏置自动计算，确保零位对齐

### Sim2Real：仿真Hinge角 → 真机舵机（控制）

```
RL网络输出 [θ_rel, ...]
    ↓ [Step 1: 零位反校准]
      (加上初始HOME位置的偏置)
仿真Hinge角 [θOA_x_r, θOA_y_r, ...] (弧度)
    ↓ [Step 2: 逆坐标系映射]
      (VirtualHingeMapper 逆向应用偏置)
方向向量 [OA_right, OB_right, OA_left, OB_left]
    ↓ [Step 3: IK反解]
      (向量 × 臂长 → 末端坐标 → IK求解舵机角)
真机舵机 [θ51, θ52, θ53, θ54, θ61, θ62, θ63, θ64]°
    ↓ [可选: Bank A/B映射]
      (只在发送硬件指令前应用，RL层禁用)
硬件指令 → 真机执行
```

**关键点**：
- IK 选择最近的解确保连续性 (prev_state同步)
- Bank A/B映射在硬件层应用，转换层禁用
- 转换延迟 < 5ms (FK + IK)

## 快速使用

### 1. 基础初始化

```python
from mujoco_playground._src.convert.sim2real_bridge import RealRobotBridge
import numpy as np

# 初始化转换桥梁
bridge = RealRobotBridge()
```

### 2. 观测转换 (Real2Sim)

```python
# 真机舵机读数 (度数)
servo_pos = np.array([260, 170, 170, 260, 260, 170, 170, 260])
#                    [M51  M52  M53  M54  M61  M62  M63  M64]

# 转换为仿真Hinge角 (弧度，相对于零位)
hinge_obs = bridge.servo_angles_to_hinge_angles(servo_pos)
# 返回: [OA_x_r, OA_y_r, OB_x_r, OB_y_r, OA_x_l, OA_y_l, OB_x_l, OB_y_l]
```

### 3. 控制转换 (Sim2Real)

```python
# RL 输出的 Hinge 角 (弧度，相对于零位)
hinge_action = np.array([0.1, 0.05, 0.1, 0.05, 0, 0, 0, 0])

# 转换为真机舵机指令 (度数)
servo_cmd = bridge.hinge_angles_to_servo_angles(
    hinge_action,
    apply_bank_ab=False  # RL层禁用Bank A/B，硬件层启用
)
# 返回: [θ51, θ52, θ53, θ54, θ61, θ62, θ63, θ64]
```

### 4. 集成到RL框架

**舵机到网络 观测适配器**：
```python
class RealObservationAdapter:
    def __init__(self):
        self.bridge = RealRobotBridge()
    
    def get_hinge_angles(self, servo_positions):
        return self.bridge.servo_angles_to_hinge_angles(servo_positions)
```

**控制指令下发 网络输出到舵机**：
```python
class TricoPolicy:
    def __init__(self):
        self.bridge = RealRobotBridge()
    
    def execute_action(self, hinge_action):
        servo_cmd = self.bridge.hinge_angles_to_servo_angles(
            hinge_action,
            apply_bank_ab=False
        )
        return servo_cmd
```

## 关键设计原则

1. **坐标系统一**
   - 所有计算在 `viz_base_frame` 本地坐标系
   - OA/OB 偏置由 VirtualHingeMapper 自动处理

2. **方向向量中介**
   - 不直接对比真机角和仿真角 (单位、范围、坐标系都不同)
   - 所有转换经过方向向量这个中间表示

3. **IK 连续性保证**
   - 初始化时同步 prev_state 到 HOME
   - 观测时同步 prev_state 到当前舵机角
   - 确保 IK 选择最近的解，避免跳变

4. **Bank A/B 解耦**
   - RL/转换层：禁用 (`apply_bank_ab=False`)
   - 硬件层：启用 (底层驱动处理)

## Sim2Real 转换模块

这个目录是 **RL 与真机通信的唯一通道**，包含完整的仿真↔真机数据转换功能和调试工具。



## 🔍 核心方法

### `servo_angles_to_hinge_angles(servo_angles_8d)`
**功能**：真机舵机角 (度) → 仿真Hinge角 (弧度)

```python
servo_8d = np.array([260, 170, 170, 260, 260, 170, 170, 260])
hinge_8d = bridge.servo_angles_to_hinge_angles(servo_8d)
# 返回: [OA_x_r, OA_y_r, OB_x_r, OB_y_r, OA_x_l, OA_y_l, OB_x_l, OB_y_l]
```

### `hinge_angles_to_servo_angles(hinge_angles_8d, apply_bank_ab=False)`
**功能**：仿真Hinge角 (弧度) → 真机舵机角 (度)

```python
hinge_8d = np.array([0.1, 0.05, 0.1, 0.05, 0, 0, 0, 0])
servo_8d = bridge.hinge_angles_to_servo_angles(hinge_8d, apply_bank_ab=False)
# 返回: [θ51, θ52, θ53, θ54, θ61, θ62, θ63, θ64]
```
